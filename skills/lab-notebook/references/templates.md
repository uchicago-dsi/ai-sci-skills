# Notebook Templates

## One-File Pattern

Use this when the project keeps one main notebook:

- Current target
- Current understanding
- Success criterion
- Leading hypotheses or current best branch
- Next best checks
- Chronology

Each chronology entry should contain:

- Question or goal
- Action taken
- Evidence
- Representative artifact or plot path when visual evidence mattered
- Result
- Conclusion, if the entry is drawing an inference
- Inference confidence: `low`, `medium`, or `high`
- Decision impact / next step

## Two-Layer Pattern

Use this when the project already separates campaign detail from lab-wide summary:

- Detailed campaign notebook:
  - full chronology
  - exact run ledger
  - run-by-run evidence
- Sparse lab notebook:
  - current best explanation
  - confidence in that explanation when it materially affects the next decision
  - strongest wins and strongest negatives
  - decision changes
  - links to detailed notebook and artifacts

## Summary Update Triggers

Update the summary when:

- the current best branch changes
- a new negative result closes a family of ideas
- the next discriminative check changes
- a bug or operator family is ruled in or out
- the project should stop pursuing a line of work

## Bad Entries

- raw command dump with no result
- result with no evidence path
- metric list with no interpretation
- inference left implicit when the next step depends on it
- interpretation with no consequence
